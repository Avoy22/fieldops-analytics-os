# FieldOps Analytics OS — Project Plan

## Problem

Marketplace companies need to understand revenue, work-order performance, buyer behavior, provider performance, and payment delays.

Raw operational data is often spread across work orders, payments, buyers, providers, and support systems. This project turns that data into business insights.

## Target User

- Data analyst
- Business intelligence analyst
- Finance analyst
- Operations manager
- Marketplace growth team
- Customer success team

## Core Business Questions

1. How much gross work order value is generated each month?
2. How much platform revenue is earned?
3. What is the platform take rate?
4. Which buyers generate the most revenue?
5. Which providers complete the most work?
6. Which categories have the highest revenue?
7. Which work orders are delayed or cancelled?
8. Which buyers have late payment behavior?
9. Which locations have strong or weak marketplace performance?

## MVP Features

- Synthetic marketplace dataset
- SQL analysis queries
- Finance KPI dashboard
- Work order health dashboard
- Payment delay dashboard
- Business insight report

## Final Portfolio Outcome

A complete analytics case study that demonstrates SQL, Python, dashboarding, business intelligence, and finance analytics skills.
